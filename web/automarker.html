Automated Testing

9 marks of the assignment will awarded by an automated marker. In order to help students avoid problems with the automarker (e.g. scoring 0 for a silly mistake), the automarker will run at least every 6 hours during the last week until the assignment is due. Code submitted with give will be automarked within the following 6 hours and the results will be emailed back to you.

We reserve the right to refine the automarker over time, but if your code passes the automarker at present, then you can be reasonably sure you haven't made too many silly mistakes.

The automarker performs several types of tests. Each of these are worth some portion of the automarker's results. Examples are:

  • compare raw output from dictCompletions() and/or dictLookup()
  • examine your trie data structure in detail to ensure all required nodes exist and no unnecessary nodes exist.
  • test with valgrind to check for memory leaks (See Debugging Tools)
  • test with mudflap to find memory usage errors (See Debugging Tools)
